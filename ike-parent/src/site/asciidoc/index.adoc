= IKE Parent POM

Centralized dependency version management and AsciiDoc toolchain
configuration for IKE Community projects.
Inherit from `ike-parent` to get aligned artifact versions
and the full multi-renderer documentation pipeline
without declaring explicit versions on each dependency.

== Usage

Set `ike-parent` as your project's parent:

[source,xml]
----
<parent>
    <groupId>network.ike.platform</groupId>
    <artifactId>ike-parent</artifactId>
    <version>1</version>
</parent>
----

Then declare dependencies without version:

[source,xml]
----
<dependency>
    <groupId>network.ike.docs</groupId>
    <artifactId>koncept-asciidoc-extension</artifactId>
</dependency>
----

== Managed Artifacts

`ike-parent` manages versions for all IKE artifacts inline
via its `<dependencyManagement>` section:

- `network.ike.tooling:ike-build-standards`
- `network.ike.docs:ike-doc-resources`
- `network.ike.docs:minimal-fonts`
- `network.ike.docs:docbook-xsl`
- `network.ike.docs:koncept-asciidoc-extension`
- `network.ike.docs:semantic-linebreak`

External consumers can also import the BOM
(`network.ike.platform:ike-bom`) for version alignment
without inheriting `ike-parent`'s build conventions.
